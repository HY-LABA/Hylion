## 테스트 목표

`hil_inference.py` (TODO-07 산출물) 를 Orin + 실 follower SO-ARM 1대 + 카메라 2대(top / wrist) 환경에서 실행.

- **dry-run** 모드: 카메라 프레임 + joint state 를 받아 forward → action 을 stdout/JSON 에 dump (팔 미동작)
- **live** 모드: 5 step → 50 step 순서로 팔에 action 을 실제 송신하여 안전 유지 확인

본 검증의 목적은 "파이프라인이 하드웨어까지 이어져 동작하는지" 확인이지 "태스크 성공(큐브 집기)" 이 아님.

## DOD (완료 조건)

1. dry-run 5 step — exit code 0, `/tmp/hil_dryrun.json` 저장, **팔 미동작** 확인
2. dry-run JSON action range — 비정상적으로 큰 값(절댓값 수십 이상) 없음
3. live 5 step — 팔이 짧게 움직이고 멈춤, 관절 한계·충돌·과전류 음 미발생
4. live 50 step — step 3 정상 시, 팔 동작 지속·안전 유지
5. `docs/storage/07_smolvla_base_test_results.md` §7 작성 완료 (7.1~7.4)

## 환경

- Orin JetPack 6.2 | Python 3.10 | venv `~/smolvla/orin/.hylion_arm`
- 접근: devPC → `ssh orin` → Orin (`laba@ubuntu`)
- **필요 하드웨어**: follower SO-ARM 1대 (`/dev/ttyACM*`) + 카메라 2대(top / wrist)
- 안전 장치: SO100Follower 기본 토크 한계 + `n_action_steps=5` (기본값) + `--max-steps` 상한
