### Study 목적

- R2 마이그레이션 영향 영역 (file·line 수준) 식별

### 본 프로젝트 SO-101 하드웨어 fact (02_hardware §6 인용)

| 항목 | 값 |
|---|---|
| 키트 등급 | **Arm Kit Pro** |
| Follower 모터 | STS3215 **12V**, 1/345 gear (C018 또는 C047) × 6 |
| Leader 모터 | STS3215 **7.4V**, 혼합 (C044 × 2 / C001 × 1 / C046 × 3) |
| Follower 전원 | DC 12V |
| Leader 전원 | DC 5V (보드 공급) |
| 보유 수량 | follower 1개 + leader 1개 (1쌍) |

SO-101 follower 는 12V 고토크 구성, leader 는 7.4V 저전압 구성이라 하드웨어 등급이 다름.
소프트웨어 계층(lerobot)에서 이 차이를 어떻게 다루는지가 본 study 의 핵심.

## §3 모터 사양·gear ratio 차이

### upstream 코드 관점

`SOFollower.__init__` 에서 모터 초기화:

```python
self.bus = FeetechMotorsBus(
    port=self.config.port,
    motors={
        "shoulder_pan": Motor(1, "sts3215", norm_mode_body),
        "shoulder_lift": Motor(2, "sts3215", norm_mode_body),
        "elbow_flex":   Motor(3, "sts3215", norm_mode_body),
        "wrist_flex":   Motor(4, "sts3215", norm_mode_body),
        "wrist_roll":   Motor(5, "sts3215", norm_mode_body),
        "gripper":      Motor(6, "sts3215", MotorNormMode.RANGE_0_100),
    },
    ...
)
```

모터 타입은 SO-100/101 모두 `"sts3215"` 로 동일. **SO-100 vs SO-101 branch 없음**.

### 하드웨어 실제 차이 (02_hardware §6 기준)

| | SO-100 (구형 — 참조용) | SO-101 / Arm Kit Pro (본 프로젝트) |
|---|---|---|
| Follower 전압 | 5V (구성 불명) | **12V** |
| Follower gear | 불명 | 1/345 (C018/C047) |
| Leader 전압 | 5V (구성 불명) | **7.4V** |
| Leader gear | 불명 | 혼합 (1/191·1/345·1/147) |

**소프트웨어 계층에서 gear ratio 차이를 별도 처리하지 않는다.** lerobot 은 모터 `id`·`type="sts3215"` 와 `calibration JSON` 으로 실제 동작 범위를 정의하며, gear ratio 는 calibration 절차(`set_half_turn_homings` + `record_ranges_of_motion`) 에서 흡수된다.

**결론: 모터 사양·gear ratio 차이가 lerobot 코드 수준에서 SO100 vs SO101 분기 없음. 동일.**

## §4 Calibration·motor_id 차이

### motor_id 매핑

SOFollower·SOLeader 모두 모터 ID 1~6 고정:

```
shoulder_pan=1, shoulder_lift=2, elbow_flex=3,
wrist_flex=4, wrist_roll=5, gripper=6
```

SO-100 / SO-101 구분 없이 동일 매핑.

### calibration

SOFollower.calibrate() 절차:

1. `disable_torque()` — 전체 모터 해제
2. `set_half_turn_homings()` — 중간 범위 기준점 설정
3. `record_ranges_of_motion(unknown_range_motors)` — 각 조인트 범위 기록
4. `wrist_roll` 은 full turn (0 ~ 4095) 고정
5. 결과를 `MotorCalibration(id, drive_mode, homing_offset, range_min, range_max)` 로 저장

SO-100 / SO-101 분기 없음. `~/.cache/huggingface/lerobot/calibration/{robots,teleoperators}/{type}/{id}.json` 경로도 `type` 문자열 (`so100_follower` vs `so101_follower`) 에 따라 달라진다는 점이 **유일한 실질적 차이**.

**결론: calibration 절차 동일. 단 calibration JSON 저장 경로가 `type` 문자열 의존 → SO101 로 전환 시 기존 SO100 calibration JSON 을 새 경로에 복사 또는 재calibration 필요.**

## §5 robot_type 문자열 차이 + lerobot 내부 dispatch

### 등록 구조 (draccus ChoiceRegistry)

`config_so_follower.py`:
```python
@RobotConfig.register_subclass("so101_follower")
@RobotConfig.register_subclass("so100_follower")
class SOFollowerRobotConfig(...)
```

두 robot_type 문자열이 **동일 config 클래스에 이중 등록**. CLI / YAML 에서 어느 문자열을 써도 같은 클래스로 dispatch 됨.

### make_robot_from_config / make_teleoperator_from_config (orin/lerobot/robots/utils.py, teleoperators/utils.py)

현재 orin 코드는 두 분기를 모두 처리:

```python
# robots/utils.py
elif config.type == "so100_follower":
    from .so_follower import SO100Follower
    return SO100Follower(config)
elif config.type == "so101_follower":
    from .so_follower import SO101Follower
    return SO101Follower(config)
```

`SO100Follower` 와 `SO101Follower` 가 동일 클래스이므로, 두 분기의 실 동작은 동일.

### calibration 경로 영향

```
so100_follower → ~/.cache/huggingface/lerobot/calibration/robots/so100_follower/{id}.json
so101_follower → ~/.cache/huggingface/lerobot/calibration/robots/so101_follower/{id}.json
```

robot_type 문자열 변경 시 calibration 파일 경로가 달라지므로, 기존 calibration JSON 이 있으면 이전 또는 재실행 필요.

## §6 본 프로젝트 채택 결정

### 결정: SO101Follower / SO101Leader + robot_type "so101_follower" / "so101_leader"

**근거**:
1. 본 프로젝트는 **SO-101 Arm Kit Pro** 물리 하드웨어를 사용 (02_hardware §6 확인).
2. SO-101 은 SO-100 의 후속 모델 (lerobot 공식 upstream 도 `SO100/101` 이중 등록).
3. lerobot 코드 수준에서 두 클래스는 동일 — 인터페이스 호환성 완전 보장.
4. 데이터셋 `robot_type` 메타데이터, calibration 경로, CLI 인자 모두 하드웨어와 일치하는 이름을 쓰는 것이 추적·유지보수 측면에서 우월.
5. lerobot 공식 examples·docs 가 SO-101 을 신형 기준으로 다루는 방향.

**단, 다음은 변경하지 않는다**:
- 데이터셋 ID `lerobot/svla_so100_pickplace` — 외부 HF Hub 자원, 이름 변경 X
- `orin/lerobot/` 내부 utils.py 의 `so100_follower` / `so100_leader` 분기 — 하위 호환성 유지 (삭제 X, SO101 분기 추가됨)
