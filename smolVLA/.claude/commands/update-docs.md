# /update-docs

현재 파일 구조를 파악해서 navigator 문서들을 최신 상태로 업데이트한다.
코드 내용을 읽을 필요 없이 파일/디렉토리 구조만 기반으로 한다.

## 실행 순서

1. 프로젝트 전체 파일 구조를 파악한다 (depth 3 수준).
   - `orin/` 하위 디렉토리 및 주요 파일
   - `docs/` 하위 구조
   - `scripts/` 하위 파일
   - 루트 레벨 파일들

2. navigator 문서 대상을 찾는다:
   - 루트 `README.md` (없으면 생성 여부 확인)
   - `docs/` 하위 `*overview*`, `*structure*`, `*navigator*` 패턴 파일
   - 각 주요 디렉토리의 `README.md`

3. 각 navigator 문서를 현재 파일 구조 기준으로 업데이트한다:
   - 존재하지 않는 경로 제거
   - 새로 생긴 파일/디렉토리 추가
   - 구조 설명이 현재 상태와 다른 부분 수정
   - 코드 내용 추측이나 설명 추가는 하지 않는다 — 구조만 반영

4. 완료 후 보고:
   - 업데이트한 파일 목록
   - 추가/제거된 항목
   - 확인이 필요한 사항 (구조 설명이 모호한 부분 등)
